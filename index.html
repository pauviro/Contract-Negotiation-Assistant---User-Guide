Understanding Recommended Actions

Each deviation receives one of these recommended actions.

Action Meaning What to Do
❌ Reject Playbook says "Not acceptable" Push back firmly. Use Playbook citation.
✅ Potentially Acceptable Playbook has a fallback Negotiate for better terms before accepting. The fallback is the minimum, not the target.
⚠️ For Consideration No Playbook guidance, but precedents exist Consult Legal POC or expert before deciding.
➡️ Route to POC No guidance anywhere Contact the Expert POC identified by the Assistant.
🚫 Auto-Reject Supplier weakened an obligation (best-effort) Reject — this removes enforcement rights.

Key Rules

Important principles to keep in mind.

You always confirm. The Assistant will never generate a document without asking you first. You see the summary and approve.

Comments ≠ Deviations. Only actual text changes count as deviations. Comments provide context only.

Playbook always wins. Even if 10 contracts accepted a deviation, if the Playbook says "No" → the recommendation is Reject.

Consistent IDs across rounds. Deviation #5 in Round 1 stays #5 in Round 2, Round 3, and Final.

No invented language. The Assistant retrieves information from approved sources only. It does not create legal text or make final decisions.

Phase 1

Round 1 — Initial Analysis

Round 1 — Analyze the attached redline against the Base Template. Complete the ENTIRE analysis before presenting anything to me — do NOT stop to ask how to proceed, do NOT present partial results. Read EVERY section word by word (FWA Cl.1-18 + Schedule 1, ITC §1-28 Schedule 2, Exhibits 1-6, Site T&Cs Schedule 3). Any text that differs from the Base Template is a Track A deviation. Comments are Track B only. Apply Track A Gate (Q1-Q4), then L1 Playbook and L2 Clause Repository for every Track A deviation. Take as long as needed — I will wait. Only present results when the FULL analysis is complete: 1) Document Registration, 2) Negotiation Difficulty Assessment, 3) Executive Summary + Complete Deviation List, 4) Completeness Verification. Then ask before generating document.
Phase 2

Intermediate Round

New round — I'm uploading the new redline AND the previous round's analysis document. Read the previous analysis FIRST as baseline. Then compare the new document word by word against the Base Template to identify all Track A deviations. Also compare against previous round to generate Summary of Movement (resolved, new, modified, unchanged). Apply L1 Playbook and L2 Clause Repository for each. Maintain same deviation IDs from previous round. Present: 1) Summary of Movement, 2) Executive Summary with Open/Closed, 3) Complete Deviation List with status, 4) Completeness Verification. Then ask before generating document.
Phase 3

Final Version

Final Version — I'm uploading the clean/definitive contract AND the previous round's analysis document. Read the previous analysis FIRST as baseline. Then compare the final contract word by word against the Base Template to confirm all deviations are resolved. Assign Outcomes to every deviation (Amazon Win if text matches template, Bilateral Compromise if partially modified, Concession if supplier's position fully accepted, Technical if non-substantive). All deviations must show ✔ Resolved. Present: 1) Document Registration, 2) Negotiation Timeline, 3) Final Negotiation Scorecard, 4) Summary of Movement to Final, 5) Executive Summary Action Table, 6) Completeness Verification. Then ask before generating document.

Tips for Best Results

Maximize accuracy and performance.

📄

Always use .docx — PDF significantly reduces analysis accuracy.

✏️

Keep tracked changes visible — the Assistant needs markup to identify supplier modifications.

💬

Don't resolve comments — they provide important context for the analysis.

🔄

One contract per chat — start a new chat for each supplier or contract.

🔗

Upload previous analysis for Round 2+ — this ensures continuity between rounds.

🔢

Check deviation count — if it seems too low, ask the Assistant to re-review section by section.

Frequently Asked Questions

Common questions about the Assistant.

Can I upload a PDF instead of .docx?
No. PDF files significantly reduce accuracy. Always use .docx format with tracked changes visible.
What if I already accepted the tracked changes?
The Assistant needs to see the markup to identify supplier modifications. Save a new version with Track Changes visible before uploading.
Do I need to upload the Base Template, Playbook, or Clause Repository?
No. These documents are already connected to the Assistant. No upload needed.
Can I analyze multiple contracts in the same chat?
No. Start a new chat for each supplier or contract to avoid confusion and maintain accuracy.
What if the deviation count seems wrong?
Ask the Assistant: "Review the document again in detail, section by section, and check if you missed any deviations."
Does the Assistant make final decisions?
No. The Assistant provides recommendations based on approved sources. You always review and approve before any action is taken.
Can I use this for non-3PL contracts?
The Assistant is currently configured for 3PL contracts using the connected Base Template, Playbook, and Clause Repository. However, the underlying logic is adaptable — as long as you have a Base Template (your standard contract) and a Clause Repository (historical precedents with approved deviations), the agent can be configured to support any contract type. Contact Pau to discuss setup for other categories.
